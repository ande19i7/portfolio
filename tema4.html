(1/2) Assetliste og designdokumentation

Opgave: 04.01.05 assetliste og designdokumentation
Projekt type: Kodning / Grafik / Tegning / SVG
Værktøjer: HTML / Adobe Illustrator / Adobe XD / Adobe Color
Individuel/gruppe opgave: Individuel

Opgaveformulering

I denne opgave skulle man færdiggøre designet til eget spil. Det var påkrævet at rentegne sin baggrund, slutskærme og sine UI-elementer, og derefter aflevere det som en komplet assetliste. Herefter skulle man færdiggøre sin designdokumentation.

Krav

  • Grafik-elementer skal eksporteres fra Adobe Illustrator til SVG-filer
  • Farver, typografi, tekstopsætning og former på knapper skal passe til det visuelle udtryk
  • Designdokumentation afleveres i PDF med link til assetlisten

Proces

Første del af temaet gik hovedsagligt ud på, at lære at analysere spildesign og hvordan spil engagerer med brugeren gennem teori og diverse konventioner inden for UI-elementer, figurelementer, baggrundsdesign og kompositionsteknikker. Derudover fik man generel viden omkring filformater, og særligt formatet "SVG", som typisk anvendes til eksportering af de grafiske elementer til spil.
Med fokus på disse konventioner, blev eget spildesign udviklet og grafikken blev dernæst rentegnet i Adobe Illustrator. I forhold til andre programmer, er Adobe Illustrator yderst effektiv til at arbejde med grafik, og fungerer godt når man skal eksportere SVG-filer, som var tilfældet i denne opgave.
Med det konkrete indhold af spillet og de grafiske elementer på plads, blev det hele samlet i en desindokumentations-rapport, som fungerede som et slags udvidet style-tile. På baggrund af dette havde man et godt fundament til skabelsen af det endelige spil.

(2/2) Individuelt spil

Opgave: 04.05.01 individuelt spil
Projekt type:Kodning / Animation / Grafik / Tegning / Lydhåndtering / Spil
Værktøjer: HTML / CSS / JS / Adobe Illustrator / Adobe Auditon / Voice Record 7
Individuel/gruppe opgave: Individuel

Opgaveformulering

I denne opgave skulle man færdiggøre sit spil. Det var påkrævet at følge kravene til animationer, lyd, implementering og aflevering. Derudover at følge kravene til struktur, design, dokumentation etc. som kunne findes i delafleveringer op til. Skal afleveres i PDF med link til spillet og designdokumentation med aktivitetsdiagram og State Machine Diagram.

Krav

  • Der må ikke benyttes gif-animation, videoklip, After Effects mm.
  • Lydeffekter og tale produceres selv
  • Spillet skal bygges op af HTML, CSS, JS og SVG

Proces

Det primære formål i den sidste del af grundlæggende animation, var primært at lære om programmeringssproget JavaScript i kombination med CSS-animationer, der sammen med UI-elementer blev brugt til at kode mere dynamiske og avancerede features (ens spil). Derudover fik man kendskab til Adobe Audition, der gjorde det muligt at personliggøre sit spil med lydeffekter og underlægningsmusik.

Det færdige spil:

Aktivitets - og State Machine Diagram fik man kendskab til undervejs i temaet, hvilket i bund og grund beskriver en aktivitet bestående af detaljerede handlinger og beslutninger. I spillet sker der ting i en bestemt rækkefølge, men hvor det ikke altid er det samme der sker hver gang aktiviteten udføres. Her er diagrammerne gode til holde overblik over aktiviteterne. State Machine Diagram er det mere detaljeret diagram, der samler flere aktiviteter i hver deres "states". Netop disse værktøjer / diagrammer blev brugt til at planlægge det interaktive flow i spillet, og dernæst kunne man begynde at programmere sit spil vha. JavaScript, HTML og CSS. Uden indlæringen af JavaScript, ville det ikke være muligt at gøre spillet interaktivt, da der ikke vil ske en "aktivitet" når man trykker på nogle af grafik-elementerne. Derfor er kombinationen af CSS animation og JavaScript afgørende ift. at lave et spil.
Til sidst blev der tilføjet yderligere materiale til designdokumentationen, hvor bl.a. aktivitetsdiagram og State Machine Diagram var inkluderet.
